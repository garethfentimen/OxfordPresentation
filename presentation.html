Making administrating Research Simpler

Before the Project was paused, the UI side of the project was 70% complete, with only 4 bugs outstanding and with only one known regression. All in less than 4 sprints...

How was that possible?

How was the project able to gain so much momentum in such a small space of time?

Continuous Integration

  • When a commit is made to a particular branch a project "build" is started. This branch could be just the general dev branch or it could be a dedicated release branch
  • The source code is delivered to a "build machine" to be built and unit tested from scratch
  • This sounds like a small gain, but how many times have you committed code that doesn't work on a colleagues machine or on the Test server?
  • I know I certainly have many times ;) more about this later..

Continuous Deployment

When a commit is made and the resulting build process is successful then the Test Environmetn owner is asked whether it is OK to update the environment with the new change.
