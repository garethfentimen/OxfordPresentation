The MARS Project

Making administrating Research Simpler

The UI for the MARS project was 70% complete, with only 4 bugs outstanding and with only one known regression. In less than 4 sprints...

How was the project able to gain so much momentum in such a small space of time?

1. Continuous Integration

  • Project Builds Automatically
  • General/Development branch and Dedicated release branch
  • Source code is delivered to "build machine" to be built and unit tested

2. Continuous Deployment

  • Gatekeeping process
  • Automatic Deployment

3. Using The Dependency Injection Design Pattern Using Constructor-based Injection

  • Coupling between classes is explicitly made during the web project startup routine

  • prefer composition over inheritance when we share dependent code

  • principal of least knowledge

3. Using The Dependency Injection Design Pattern Using Constructor-based Injection Continued...

Other benefits

  • Easier to unit test our business logic, interface
  • New keyword reduces the temptation to do too much in the constructor

Problems with the DI pattern

  • Busy constructors in classes and controllers which grow
  • Tooo many classes!

5. Maintain Good Code Conventions - General Points

  • Classes are capitalised, function parameters are lowercase, just always
  • Boy scouting and the "don't be part of the problem" attitude
  • DRY - Try not mindlessly copy and paste code
  • Make the Code, particularly in Controllers, read and flow well...
  • Avoid Pointless abbreviations...
