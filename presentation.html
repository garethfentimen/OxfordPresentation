Continuous Integration

When a commit is made a project "build" is kicked off

This means that the source code is delivered to a machine (could be a Cloud based Virtual Machine) and rebuilt from scratch

This sounds like a small gain, but how many times have you committed code that doesn't work on a colleagues machine or worse on the Test server - I know I certainly have many times ;) more about this later..

The Dependency Injection Design Pattern Using Constructor-based Injection

This means that the coupling between classes is explicitly made during the web project startup routine

This helps us to "prefer composition over inheritance" when we share dependent code. It also helps us to keep to the principal of least knowledge

It also has another beneficial side effect - it makes it easier to unit test the business logic
